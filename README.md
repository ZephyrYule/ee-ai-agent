#  EE-AI-Agent-50Days
**深圳大学电子信息工程 | 面向物理世界的 AI Agent 实习实战项目库**

##  项目简介
本项目记录了我从 0 到 1 构建跨界 AI 能力的 50 天硬核实战全过程。作为 EE（电子信息工程）背景的开发者，我致力于打破“纯软件 Agent”的边界，**将大模型（LLM）的云端算力与物理硬件、本地工作流深度结合**。

本项目放弃低代码拖拽平台，全程采用纯 Python 手写代码，聚焦于 Tool Use（函数调用）、多 Agent 协同与自动化工作流编排，旨在打造真正能落地商业场景（如智能硬件控制、研发效能提升）的实用型 AI Agent。

##  核心落地项目 (Projects)

###  1. 智能数字万用表 Agent（状态：已跑通）
- **场景**：打破大模型“缸中之脑”的限制，赋予云端 AI 物理探测能力。
- **核心逻辑**：基于阿里云通义大模型，手写 `Function Calling` 协议，使大模型能够根据自然语言指令（如“板子短路了，测一下 VCC 引脚”），精准提取参数并触发本地 Python 中断，调用底层 ADC 测量函数并回传真实数据。
- **技术栈**：Python, OpenAI SDK (兼容模式), JSON 数据流处理, Prompt Engineering。

### ⚙️ 2. 创客硬件智能选型与诊断 Agent（开发中）
- **场景**：面向智能硬件创客生态（如 xTool 等应用场景）。
- **核心逻辑**：构建带记忆与逻辑推理的多轮对话 Agent。输入硬件故障现象或设计需求（如“步进电机发热严重”或“需要选一个 5V 转 3.3V 的 LDO”），Agent 自动调取本地知识库，输出三步排查法或元件选型推荐。
- **技术栈**：RAG（检索增强生成）, 向量数据库基础应用, 本地文件系统交互。

### 3. 研发效能与本地代码自动化 Review Agent（规划中）
- **场景**：面向 AI 研发团队内部效能提升（如 YouWare 等应用场景）。
- **核心逻辑**：联动 Cursor 等新一代 AI 编码工具理念，编写本地脚本。让 Agent 能够自动读取指定 `.py` 文件的代码结构，排查语法错误、变量拼写问题，并输出格式化的诊断报告。
- **技术栈**：Python OS 模块交互, 大模型结构化输出调优, 自动化工作流编排。

## 技术栈与开发环境 (Tech Stack)
- **底层驱动**：Python 3.14 (全手工编写代码，无依赖低代码平台)
- **AI 算力**：阿里云百炼平台 (Qwen-Plus/Max)
- **通信协议**：基于 OpenAI 标准协议的 API 深度调用
- **版本控制**：Git & GitHub (遵循标准 Commit 规范)
- **开发环境**：VS Code 本地部署环境
